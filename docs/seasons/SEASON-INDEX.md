## Overview

GoNode development is organised into seasons, each lasting approximately 6 months. Each season has clear goals, deliverables, and a post-mortem document describing what was achieved, what was learned, and what remains for the next season. This index links to all season documentation as the project progresses.

The season structure matches the phase-based roadmap from [ROADMAP.md](../ROADMAP.md) but provides more granular documentation of actual execution versus planned deliverables. Seasons are retrospective documents, written after the fact, while the roadmap is a forward-looking plan.

| Season | Timeline | Theme | Status |
|:-------|:---------|:------|:-------|
| Season 1 | April 2026 - July 2026 | Foundation, grants, documentation | In progress |
| Season 2 | July 2026 - January 2027 | Permissioned testnet, first revenue | Planned |
| Season 3 | January 2027 - July 2027 | Public testnet, MiCA filing | Planned |
| Season 4 | July 2027 - January 2028 | TGE preparation, audits | Planned |
| Season 5 | January 2028 - July 2028 | Mainnet launch, token generation | Planned |
| Season 6+ | July 2028 onwards | DAO transition, ecosystem expansion | Planned |

## Season 1: Foundation (April 2026 - July 2026)

**Theme:** Build the legal, documentary, and financial foundation for GoNode.

**Phase mapping:** Phase 0 of the roadmap.

**Primary goals:**

- Establish Swiss Foundation (Stiftung) for protocol operations
- Secure initial non-dilutive funding through grants
- Publish complete concept documentation
- Build core team to 3+ FTE
- Commission legal opinion from qualified German law firm

**Key deliverables:**

| Deliverable | Status |
|:------------|:-------|
| README and concept documentation | Complete (this document set) |
| Architecture and security document | Complete |
| Tokenomics specification | Complete |
| Business model and financial projections | Complete |
| Legal compliance analysis | Complete |
| Roadmap and phase planning | Complete |
| Wire protocol specification | Complete |
| Smart contract specifications | Complete |
| Foundation incorporation | In progress |
| First grants secured | In progress (NLnet, EF, Arbitrum pipeline) |
| Core team hiring | In progress |

**Post-mortem:** [Season 1 retrospective](SEASON-1.md) (published at end of season, approximately July 2026)

## Season documentation format

Each season retrospective follows a standard format for consistency:

```
# Season N Retrospective

## Executive summary
  - What was the season's theme?
  - Did we achieve primary goals? (yes/partial/no)
  - Key metrics at season end

## Goals vs achievements
  - Planned deliverables with status
  - Unplanned work completed
  - Planned work deferred

## What went well
  - Technical successes
  - Organisational successes
  - Community milestones

## What went poorly
  - Setbacks and their causes
  - Missed deadlines
  - Abandoned directions

## Key learnings
  - What we would do differently
  - Knowledge gaps identified
  - Process improvements

## Metrics
  - Revenue (EUR)
  - Users (Pro subs, enterprise, AI, testnet)
  - Operators (active nodes)
  - Development (commits, PRs, contributors)
  - Community (Discord/Matrix members, newsletter subscribers)

## Financial summary
  - Revenue by stream
  - Costs by category
  - Treasury status
  - Runway remaining

## Decisions made
  - Major architectural decisions
  - Major business decisions
  - Major personnel decisions

## Goals for next season
  - Confirmed goals
  - Stretch goals
  - Contingencies
```

## Using this index

**For contributors:** Read the current season's goals and the previous season's retrospective to understand project direction.

**For auditors:** Use season documents to understand which code was developed when, and what known issues existed during each period.
